Technical Analysis and Documentation: CRAFT x PP-OCRv3 ONNXization for Full page PDF Line Text Extraction

Alexi George

June 18, 2025

Abstract

This paper presents an optimized pipeline integrating CRAFT (Character Region Awareness for Text Detection) and PP-OCRv3 (PaddlePaddle Optical Character Recognition version 3) models, converted to ONNX format, for efficient line-by-line text extraction from image-based PDF documents. The system leverages CRAFT’s robust text detection and PP-OCRv3’s enhanced text recognition capabilities, with ONNX enabling cross-platform deployment and reduced inference latency. Performance evaluations across initial and modified runs demonstrate significant improvements in detection and recognition times, making the pipeline suitable for processing complex PDF layouts in production environments.

1. Introduction

Text extraction from scanned or image-based PDFs is essential for document digitization but poses challenges due to varied layouts, multilingual text, and scanning artifacts. This work combines CRAFT, a state-of-the-art text detection model, with PP-OCRv3, an advanced OCR framework offering improved accuracy and multilingual support over its predecessors. Both models are converted to ONNX for portability and optimized inference across diverse hardware using CUDA 12. This paper describes the methodology, implementation, and performance results, highlighting optimizations that significantly reduced processing times.

2. Related Work

CRAFT (Baek et al., 2019) generates character-level heatmaps for precise text localization in complex scenes. PP-OCRv3 (PaddlePaddle, 2023) enhances OCR with optimized detection and recognition modules, supporting over 80 languages with higher accuracy (~0.92 on SynthText) than earlier versions. ONNX (ONNX, 2023) standardizes neural network representations, enabling efficient inference. Previous PDF text extraction solutions often rely on proprietary OCR or unoptimized open-source models, lacking the flexibility of ONNX-based pipelines.

3. Methodology

3.1 System Architecture

The pipeline processes PDFs as follows:

  1. PDF Preprocessing: Convert pages to images using pdf2image at 300 DPI.
  2. Text Detection (CRAFT): Identify text regions using ONNX-converted CRAFT model.
  3. Text Recognition (PP-OCRv3): Extract text from detected regions using ONNX-converted PP-OCRv3 model.
  4. Postprocessing: Group regions into lines, sorted by reading order, and output as JSON.

3.2 Model Conversion

CRAFT: Converted from PyTorch using torch.onnx.export with dynamic input shapes. Outputs include character and affinity score maps. PP-OCRv3: Converted from PaddlePaddle using paddle2onnx, with preprocessing (e.g., normalization) aligned to ONNX requirements. Models were optimized with onnx-simplifier and validated using onnx.checker. Inference was performed using ONNX Runtime with CUDA 12.

3.3 Implementation Details

Tools include PyTorch, PaddlePaddle, ONNX Runtime, pdf2image, and OpenCV. The pipeline runs on Python 3.8+ with CUDA 12 for GPU acceleration. Preprocessing normalizes images, and postprocessing applies non-maximum suppression (NMS) for bounding box filtering. PP-OCRv3’s recognition module supports multilingual text, dynamically switching models as needed.

4. Results

Performance was evaluated on full-page PDFs using ONNX Runtime with CUDA 12. The initial run used FP32 precision for both CRAFT and PP-OCRv3 models. M1 used FP16 precision for both models. M2 used FP16 precision with line clustering, determining the maximum line length by computing the maximum bounding box width across all lines on the page. Results are presented in Table 1.

Run Detection Time (s) Line Cropping Time (s) Recognition Time (s) Total Time (s)
Initial (FP32) 0.324 0.012 0.955 1.291
M1 (FP16) 0.147 0.013 0.265 0.425
M2 (FP16 + Clustering) 0.083 0.008 0.204 0.295

Table 1: Performance metrics for initial and modified runs (M1, M2) of the CRAFT x PP-OCRv3 pipeline on full-page PDFs.

Initial Run (FP32): Processed a full-page PDF with FP32 precision, with recognition dominating at 0.955 s due to numerous text regions. M1 (FP16): Used FP16 precision, reducing detection (0.147 s) and recognition (0.265 s) times by approximately 50%. M2 (FP16 + Clustering): Further optimized with FP16 and line clustering (maximum line length via maximum bounding box width), achieving 0.083 s detection and 0.204 s recognition. Expected accuracy is ~0.85 F1-score for detection (ICDAR 2015) and ~0.92 for recognition (SynthText).

5. Discussion

The results demonstrate a 77% reduction in total processing time from the initial run (1.291 s) to M2 (0.295 s), attributed to FP16 precision and line clustering optimizations on full-page PDFs. FP16 reduced computational overhead, while clustering by maximum bounding box width streamlined line grouping. PP-OCRv3’s enhanced recognition module improved accuracy for multilingual text compared to earlier PPOCR versions. However, challenges remain with low-resolution PDFs, complex layouts (e.g., tables), and dynamic model switching for multilingual support. ONNX conversion may introduce minor accuracy loss (~1–2%), which is acceptable for most applications.

6. Conclusion

The CRAFT x PP-OCRv3 ONNX pipeline provides an efficient, portable solution for line-level text extraction from image-based PDFs. Significant latency reductions via FP16 and line clustering, combined with PP-OCRv3’s improved accuracy, make it suitable for production use. Future work includes developing an end-to-end model, supporting dynamic ONNX input shapes, and extending the pipeline for table extraction.
