Pix2Depth

A High-Fidelity Spatial Inference Pipeline for Autonomous Drone Tracking
Technical Analysis & Architectural Overview
Alexi George
April 05, 2026
Abstract: This document outlines the architectural and mathematical framework of Pix2Depth, a real-time computer vision system designed for 2D-to-3D spatial mapping. By integrating deep learning-based detection with stereoscopic depth sensors, the system achieves millimeter-accurate target localization within a user-defined Euclidean space. The following sections detail the transformation logic, hardware-in-the-loop performance, and noise mitigation strategies implemented in the current deployment.

I. Introduction

The fundamental challenge in autonomous aerial tracking is the translation of pixel-space coordinates into actionable 3D vectors. Standard monocular detection provides object identity but lacks depth perception. Pix2Depth solves this by utilizing the ZED Mini’s dual-lens configuration to generate a high-density point cloud, which is then queried using ROI centroids derived from a YOLOv11 detection head.

III. Mathematical Transformation Logic

Calibration is performed by establishing a translation vector and a rotation matrix that aligns the camera's local coordinate system with the physical orientation of the tracking environment (e.g., floor-level origin).

Notation: P_cam : Raw point (X, Y, Z) in Camera-Space P_room : Transformed point in Room-Space R : 3x3 Rotation Matrix T : Translation Vector (Origin) Computation: 1. Define Y-axis vector: V_y = P_axis - P_origin 2. Compute orthogonal X: V_x = Cross(V_y, [0, 0, 1]) 3. Compute orthogonal Z: V_z = Cross(V_x, V_y) 4. Transformation: P_room = R * (P_cam - T)

The resulting P_room provides coordinates in millimeters, where Z represents altitude above the calibrated floor, and XY represents the lateral and longitudinal position relative to the origin.

V. Technical Challenges & Robustness

A. Noise Mitigation: To prevent false positives from ceiling reflections or static background objects, the system implements a "Geometric Dead Zone" that ignores the upper 33% of the image frame.

B. Identity Persistence: When the primary YOLO detector fails (due to rapid acceleration or partial occlusion), the system triggers a Siamese Region Proposal Network (SiamRPN). This ensures the tracking loop maintains the target's identity without requiring a full detection pass every frame.
